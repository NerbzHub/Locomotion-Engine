## What public GitHub hosting means

Publishing a repository on GitHub makes it viewable and, under GitHub's Terms
of Service, may make it forkable. It does **not** grant a software licence to
reuse the code. The repository's README and `LICENSE.md` make that distinction
plainly visible.

Moving the repository to private is the strongest practical protection against
new public viewing and forks. It cannot revoke copies or forks made while the
repository was public.

## Before distributing a build

1. Keep `LICENSE.md` and the README copyright notice in the source repository
   and any source archive.
2. Review `NOTICE.md` for third-party dependency and asset obligations.
3. Do not accept external code contributions without deciding how their
   copyright ownership and permission will be recorded.
4. Consult a qualified lawyer for a bespoke commercial licence, contributor
   agreement, trademark protection, or advice specific to the project.
